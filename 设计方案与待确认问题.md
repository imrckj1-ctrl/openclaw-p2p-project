# OpenClaw P2P 插件 + 安卓 APP 设计方案

> 规则：每确认一个问题，把 `[ ]` 改成 `[x]`，并填入确认结果。

---

## 一、项目概述

做一个 OpenClaw 插件（openclaw-p2p）+ 安卓 APP，实现：
- 安卓 APP 和 OpenClaw 之间通过 WebSocket 直接通信
- 不依赖飞书/微信等第三方平台
- 支持文字消息、文件传输等

---

## 二、架构设计

```
┌─────────────┐    WebSocket    ┌──────────────────┐
│  安卓 APP    │ ◄────────────► │  openclaw-gateway │
│  (客户端)    │                │  (服务端)         │
└─────────────┘                 └────────┬─────────┘
                                         │
                                         ▼
                                ┌──────────────────┐
                                │  openclaw-p2p     │
                                │  插件             │
                                ├──────────────────┤
                                │  gateway: 收消息   │
                                │  outbound: 发消息  │
                                └────────┬─────────┘
                                         │
                                         ▼
                                ┌──────────────────┐
                                │  AI Agent 处理    │
                                └──────────────────┘
```

## 三、待确认问题

### A. 网络与连接

- [x] **A1. 连接方式**：APP 直连 OpenClaw gateway，还是通过插件自己开一个 WebSocket 服务？
  - 方案 A：复用 openclaw-gateway 已有的 WebSocket 端口（18789），插件注册为一个 channel
  - 方案 B：插件自己开一个独立的 WebSocket 端口（比如 18790）
  - 确认结果：`方案 B，独立 WebSocket 端口（18790）`

- [x] **A2. 网络环境**：APP 和 OpenClaw 在同一个局域网，还是需要跨网络（公网）？
  - 同一局域网：直连 IP 即可
  - 跨网络：需要公网 IP / 内网穿透 / 中转服务器
  - 确认结果：`家里有公网IP+域名，外出用公网IP+端口，家里用局域网IP+端口。APP 需要支持配置两个地址（家庭/外出）自动切换或手动选择。`

- [x] **A3. 认证方式**：APP 怎么证明自己是合法用户？
  - 方案 A：连接时发送 token（简单，配置文件里写死一个密钥）
  - 方案 B：用户名 + 密码登录
  - 方案 C：配对码（APP 显示一个码，OpenClaw 端输入确认）
  - 确认结果：`方案 A，连接时发 token 认证。openclaw.json 里配置 token，APP 里填写一样的。`

- [x] **A4. 断线重连**：网络断了怎么办？
  - 自动重连？间隔多久？重连几次放弃？
  - 确认结果：`自动重连，间隔 3s/5s/10s/30s 递增，不放弃（直到用户手动断开）。连接状态在 APP 顶部显示。`

- [x] **A5. 多设备**：允许多个 APP 同时连一个 OpenClaw 吗？
  - 确认结果：`单用户，第一版只支持一个设备连接`

## 六、通信协议设计

参考飞书插件的消息格式，简化为 P2P 直连版本。

### 连接握手

APP 连上 WebSocket 后，第一条消息必须是认证：

```json
// APP → 服务端：认证
{
  "type": "auth",
  "token": "openclaw.json 里配置的 token"
}

// 服务端 → APP：认证结果
{
  "type": "auth_result",
  "ok": true,
  "serverVersion": "0.1.0"
}
```

认证失败则服务端主动断开连接。

### 消息类型

#### 文字消息（APP → 服务端）

```json
{
  "type": "text",
  "msgId": "uuid-xxx",
  "content": "你好",
  "timestamp": 1715654400000
}
```

#### 图片消息（APP → 服务端）

小图直接 Base64，大图分片：

```json
// 方式一：小图直接发
{
  "type": "image",
  "msgId": "uuid-xxx",
  "fileName": "photo.jpg",
  "mimeType": "image/jpeg",
  "data": "base64编码...",
  "timestamp": 1715654400000
}

// 方式二：大图分片
// 第1步：开始
{
  "type": "image_start",
  "msgId": "uuid-xxx",
  "fileName": "photo.jpg",
  "mimeType": "image/jpeg",
  "totalSize": 2048000,
  "totalChunks": 5
}
// 第2步：逐片发送
{
  "type": "image_chunk",
  "msgId": "uuid-xxx",
  "chunkIndex": 0,
  "data": "base64..."
}
// 第3步：结束
{
  "type": "image_end",
  "msgId": "uuid-xxx"
}
```

#### 文件消息（APP → 服务端）

同图片逻辑，type 换成 `file` / `file_start` / `file_chunk` / `file_end`：

```json
{
  "type": "file",
  "msgId": "uuid-xxx",
  "fileName": "document.pdf",
  "mimeType": "application/pdf",
  "fileSize": 1024000,
  "data": "base64..."
}
```

#### AI 流式回复（服务端 → APP）

参考飞书的 streaming card 机制，分三个阶段：

```json
// 第1步：开始回复（APP 显示"AI 正在输入..."）
{
  "type": "reply_start",
  "msgId": "uuid-xxx",
  "replyTo": "用户消息的msgId"
}

// 第2步：流式内容片段（APP 追加显示）
{
  "type": "reply_chunk",
  "msgId": "uuid-xxx",
  "content": "你好，"
}
{
  "type": "reply_chunk",
  "msgId": "uuid-xxx",
  "content": "我是 AI 助手，"
}
{
  "type": "reply_chunk",
  "msgId": "uuid-xxx",
  "content": "有什么可以帮你的？"
}

// 第3步：回复结束（APP 完成渲染，启用复制等操作）
{
  "type": "reply_end",
  "msgId": "uuid-xxx",
  "fullContent": "你好，我是 AI 助手，有什么可以帮你的？"
}
```

#### 系统消息（服务端 → APP）

```json
{
  "type": "system",
  "msgId": "uuid-xxx",
  "content": "连接成功",
  "level": "info"
}
```

#### 错误消息（服务端 → APP）

```json
{
  "type": "error",
  "msgId": "uuid-xxx",
  "code": "PROCESSING_ERROR",
  "message": "处理消息时出错"
}
```

#### 斜杠命令

APP 连接成功后，自动请求命令列表：

```json
// APP → 服务端：请求命令列表
{
  "type": "get_commands"
}

// 服务端 → APP：命令列表（服务端动态下发）
{
  "type": "commands",
  "commands": [
    {
      "name": "/clear",
      "description": "清除上下文",
      "hint": "清除 AI 的对话记忆"
    },
    {
      "name": "/think",
      "description": "切换思考模式",
      "hint": "开启/关闭 AI 深度思考"
    },
    {
      "name": "/compact",
      "description": "压缩上下文",
      "hint": "压缩历史对话节省 token"
    },
    {
      "name": "/help",
      "description": "帮助",
      "hint": "显示可用命令"
    }
  ]
}
```

用户在 APP 输入框输入 `/` 时，弹出命令列表，输入越多字母越精确过滤。
选中命令后，作为普通文字消息发给服务端处理：

```json
{
  "type": "text",
  "msgId": "uuid-xxx",
  "content": "/clear",
  "timestamp": 1715654400000
}
```

#### 语音消息（APP → 服务端）

APP 端用 Android 语音识别 API 转成文字，发送文字消息：

```json
// APP 本地录音 → 语音识别 → 发送文字
{
  "type": "text",
  "msgId": "uuid-xxx",
  "content": "语音转出的文字内容",
  "source": "voice",
  "timestamp": 1715654400000
}
```

### 消息大小限制

| 类型 | 限制 |
|------|------|
| 文字 | 无限制（AI 回复可能很长）|
| 图片 | 单张 10MB，超过分片 |
| 文件 | 单个 50MB，超过分片 |
| 单片大小 | 64KB |
| 离线消息缓存 | 服务端最多 100 条 |

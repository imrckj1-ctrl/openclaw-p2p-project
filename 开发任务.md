## 一、统一 API 定义（所有任务的基础）

> 这部分是插件端和 APP 端的公共约定，两边必须严格一致。

### 1.1 WebSocket 消息协议

```typescript
// ========== 基础类型 ==========

interface BaseMessage {
  type: string;
  msgId: string;        // UUID，每条消息唯一
  timestamp: number;     // Unix 毫秒时间戳
}

// ========== APP → 服务端 ==========

// 认证（连接后第一条消息）
interface AuthMessage {
  type: 'auth';
  token: string;
}

// 文字消息
interface TextMessage extends BaseMessage {
  type: 'text';
  content: string;
  source?: 'voice';     // 来源标记：语音输入
}

// 图片消息（小图直接发）
interface ImageMessage extends BaseMessage {
  type: 'image';
  fileName: string;
  mimeType: string;      // image/jpeg, image/png 等
  data: string;          // Base64 编码
}

// 图片消息（大图分片）
interface ImageStartMessage {
  type: 'image_start';
  msgId: string;
  fileName: string;
  mimeType: string;
  totalSize: number;     // 字节数
  totalChunks: number;
}
interface ImageChunkMessage {
  type: 'image_chunk';
  msgId: string;
  chunkIndex: number;    // 从 0 开始
  data: string;          // Base64 编码
}
interface ImageEndMessage {
  type: 'image_end';
  msgId: string;
}

// 文件消息（同图片逻辑）
interface FileMessage extends BaseMessage {
  type: 'file';
  fileName: string;
  mimeType: string;
  fileSize: number;
  data: string;          // Base64 编码
}
interface FileStartMessage {
  type: 'file_start';
  msgId: string;
  fileName: string;
  mimeType: string;
  totalSize: number;
  totalChunks: number;
}
interface FileChunkMessage {
  type: 'file_chunk';
  msgId: string;
  chunkIndex: number;
  data: string;
}
interface FileEndMessage {
  type: 'file_end';
  msgId: string;
}

// 请求命令列表
interface GetCommandsMessage {
  type: 'get_commands';
}

// ========== 服务端 → APP ==========

// 认证结果
interface AuthResultMessage {
  type: 'auth_result';
  ok: boolean;
  serverVersion: string;
  reason?: string;       // 失败原因
}

// AI 回复开始
interface ReplyStartMessage {
  type: 'reply_start';
  msgId: string;
  replyTo: string;       // 用户消息的 msgId
}

// AI 回复内容片段（流式）
interface ReplyChunkMessage {
  type: 'reply_chunk';
  msgId: string;         // 同 reply_start 的 msgId
  content: string;       // 本次追加的内容
}

// AI 回复结束
interface ReplyEndMessage {
  type: 'reply_end';
  msgId: string;         // 同 reply_start 的 msgId
  fullContent: string;   // 完整回复内容
}

// 命令列表
interface CommandsMessage {
  type: 'commands';
  commands: CommandInfo[];
}
interface CommandInfo {
  name: string;          // "/clear"
  description: string;   // "清除上下文"
  hint: string;          // "清除 AI 的对话记忆"
}

// 系统消息
interface SystemMessage {
  type: 'system';
  msgId: string;
  content: string;
  level: 'info' | 'warn' | 'error';
}

// 错误消息
interface ErrorMessage {
  type: 'error';
  msgId: string;
  code: string;          // PROCESSING_ERROR, AUTH_FAILED 等
  message: string;
}

// 离线消息开始
interface OfflineMessagesStart {
  type: 'offline_messages';
  count: number;
}

// 离线消息结束
interface OfflineDone {
  type: 'offline_done';
}
```

### 1.2 消息大小限制

| 类型 | 限制 |
|------|------|
| 文字 | 无限制 |
| 图片 | 单张 10MB，超过分片 |
| 文件 | 单个 50MB，超过分片 |
| 单片大小 | 64KB (65536 字节 Base64) |
| 离线缓存 | 服务端最多 100 条 |

### 1.4 APP 端配置结构

```kotlin
// SharedPreferences 存储
data class ServerConfig(
    val name: String,           // "家里" / "外出"
    val host: String,           // "192.168.31.145" / "your.domain.com"
    val port: Int,              // 18790
    val token: String,          // 认证 token
    val useWss: Boolean = false // 是否用加密连接
)

data class AppSettings(
    val activeServerIndex: Int = 0,        // 当前选中的服务器
    val servers: List<ServerConfig>,       // 服务器列表
    val darkMode: String = "system"        // "light" / "dark" / "system"
)
```

### 1.5 数据库结构（SQLite / Room）

```sql
-- 消息表
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    msg_id TEXT UNIQUE NOT NULL,        -- 协议中的 msgId
    type TEXT NOT NULL,                 -- 'text', 'image', 'file', 'reply'
    content TEXT,                       -- 文字内容 / AI 回复内容
    file_name TEXT,                     -- 图片/文件名
    file_path TEXT,                     -- 本地存储路径
    mime_type TEXT,
    file_size INTEGER,
    is_from_user INTEGER NOT NULL,      -- 1=用户发送, 0=AI回复
    reply_to TEXT,                      -- 回复的消息 msgId
    source TEXT,                        -- 'voice' 等
    created_at INTEGER NOT NULL,        -- Unix 毫秒时间戳
    is_offline INTEGER DEFAULT 0        -- 是否离线消息
);

-- 命令缓存表
CREATE TABLE commands (
    name TEXT PRIMARY KEY,
    description TEXT,
    hint TEXT,
    updated_at INTEGER NOT NULL
);

-- 索引
CREATE INDEX idx_messages_created_at ON messages(created_at);
CREATE INDEX idx_messages_is_from_user ON messages(is_from_user);
```

  - 连接建立时记录 client，断开时清理

  - 验证：用 wscat 能连上

### T03: Token 认证
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T02
- **内容**:
  - 连接后等待第一条 `auth` 消息
  - 验证 token 是否匹配配置
  - 认证成功返回 `auth_result`，失败断开
  - 认证超时 10 秒未收到断开
  - 验证：发错误 token 被断开，发正确 token 收到 auth_result
- **产出文件**: `src/auth.js`

### T04: 文字消息收发
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T03
- **内容**:
  - 接收 APP 的 `text` 消息
  - 调用 OpenClaw agent dispatch，把消息送入 AI 处理
  - AI 回复后，通过 `outbound` 发回 APP
  - 实现 `outbound.sendText` 方法
  - 验证：APP 发文字，AI 能回复
- **产出文件**: `src/message-handler.js`, `index.js` (outbound 部分)

### T05: 流式回复
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T04
- **内容**:
  - AI 回复时先发 `reply_start`
  - 每收到一段内容发 `reply_chunk`
  - 完成后发 `reply_end`（含完整内容）
  - 如果 AI 回复过程中出错，发 `error` 消息
  - 验证：APP 能看到文字逐字出现
- **产出文件**: `src/stream-reply.js`

### T06: 图片消息接收
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T04
- **内容**:
  - 接收小图 `image` 消息，Base64 解码保存
  - 接收大图分片 `image_start` / `image_chunk` / `image_end`
  - 分片组装完成后保存文件
  - 把图片路径传给 AI（让 AI 能"看到"图片）
  - 验证：APP 发图片，AI 能描述图片内容
- **产出文件**: `src/message-handler.js` (扩展)

### T07: 文件消息接收
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T06
- **内容**:
  - 同图片逻辑，处理 `file` / `file_start` / `file_chunk` / `file_end`
  - 保存文件，把路径传给 AI
  - 验证：APP 发文件，AI 能处理
- **产出文件**: `src/message-handler.js` (扩展)

### T08: 斜杠命令
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T04
- **内容**:
  - 接收 `get_commands` 请求
  - 从 OpenClaw 的 agent 注册的命令中提取列表
  - 返回 `commands` 消息
  - 用户发送 `/xxx` 文字时，走命令处理路径
  - 验证：APP 输入 / 能看到命令列表
- **产出文件**: `src/commands.js`

### T09: 离线消息缓存
- **状态**: `[x]`
- **负责人**: `Claude`
- **依赖**: T05
- **内容**:
  - 客户端断开时，缓存 AI 回复消息（内存，最多 100 条）
  - 客户端重连认证后，检查缓存
  - 有缓存则发 `offline_messages` → 逐条补发 → `offline_done`
  - 验证：APP 断开期间 AI 有回复，重连后能收到
- **产出文件**: `src/offline-cache.js`
